Developer Pacing & Instruction Set

1. Simulated User Inputs: inp#text#

Integrates a realistic keyboard entry sequence into terminal execution. It automatically registers a 1.5s pause, types out the letters sequentially, and executes another 1.5s wait block after completion.

2. Timing Spacers: t#duration#

Inserts custom pacing delays (in seconds) anywhere inside either the Code block or the Output lines. For example, t#0.7# halts typing for exactly 0.7s before continuing, without displaying the tag itself.

3. Ordered Typing: N##code##

Wrap a chunk of the Code block in N##...## to control the order it's TYPED in, independent of where it sits in the final file. Numbered chunks type lowest-to-highest first; untagged text always types last. The cursor jumps to the right spot and existing lines shift down, just like a real editor.
